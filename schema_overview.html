Banco de Dados

O banco de dados da Plataforma Tenda é projetado para armazenar e gerenciar todas as informações necessárias para o funcionamento do sistema. Abaixo, você encontrará uma explicação detalhada de como os dados são organizados e estruturados.

Estrutura do Banco de Dados

A plataforma utiliza várias tabelas interligadas para armazenar diferentes tipos de informações. Aqui está uma visão geral das principais tabelas:

  • cidades: Armazena informações sobre as cidades, conectadas aos estados.
  • famílias: Registros de famílias, vinculados às tendas.
  • missas: Eventos de missa, vinculados às paróquias.
  • paróquias: Informações sobre paróquias, conectadas às cidades e estados.
  • padres: Registros de padres, vinculados aos perfis de usuários e paróquias.
  • funções: Funções de usuário (devAdmin, Fundador, Guardião, etc.).
  • funções_serviço: Funções de equipe/serviço (Líder, Participante, etc.).
  • etapas: Etapas de eventos ou tendas.
  • estados: Informações sobre estados.
  • variáveis_sistema: Configurações da plataforma.
  • guardiões_tenda: Guardiões de tenda, vinculados às tendas e perfis de usuários.
  • tendas: Registros de tendas (grupos), vinculados às cidades, estados, etapas e paróquias.
  • perfis_usuários: Perfis de usuários, vinculados às tendas e outros perfis (cônjuge).
  • funções_serviço_usuários: Mapeamento entre usuários e funções de serviço.

Detalhamento do Banco de Dados

Esta seção apresenta uma visão detalhada da estrutura do banco de dados, incluindo tabelas, políticas de segurança (RLS), papéis de usuário e permissões. O objetivo é explicar de forma clara como os dados são organizados, protegidos e acessados na plataforma.

Organização dos Schemas

  • auth: Gerenciamento de autenticação e usuários.
  • public: Dados principais da aplicação.
  • storage: Gerenciamento de arquivos e imagens.
  • realtime: Mensageria em tempo real.
  • vault: Gerenciamento de segredos e informações sensíveis.

Detalhamento das Tabelas

Schema: public

  • cidades
    Chave Primária: id
    Chave Estrangeira: state_id → states(id)
    Políticas RLS: Acesso total para devAdmin e Founder; leitura para Guardião, usuários autenticados e visualizadores.
  • famílias
    Chave Primária: id
    Chave Estrangeira: tenda_id → tendas(id)
    Políticas RLS: Acesso total para devAdmin e Founder; leitura para usuários autenticados e visualizadores.
  • missas
    Chave Primária: id
    Chave Estrangeira: parish_id → parishes(id)
    Políticas RLS: Acesso total para devAdmin e Founder; leitura para usuários autenticados; inserção para autenticados.
  • paróquias
    Chave Primária: id
    Políticas RLS: Acesso total para devAdmin e Founder; leitura para Guardião, usuários autenticados e visualizadores.
  • padres
    Chave Primária: id
    Chaves Estrangeiras: profile_id → users_profiles(id), parish_id → parishes(id)
    Políticas RLS: Acesso total para devAdmin e Founder; leitura para usuários autenticados e visualizadores.
  • tendas
    Chave Primária: id
    Chaves Estrangeiras: city_id → cities(id), state_id → states(id), stage_id → stages(id), parish_id → parishes(id)
    Políticas RLS: Acesso total para devAdmin e Founder; leitura para Guardião, usuários autenticados e visualizadores.
  • perfis de usuário (users_profiles)
    Chave Primária: id
    Chaves Estrangeiras: tenda_id → tendas(id), spouse_id → users_profiles(id)
    Políticas RLS: Acesso total para devAdmin e Founder; leitura para Guardião, usuários autenticados e visualizadores; atualização pelo próprio usuário.

Schema: storage

  • objects
    Chave Primária: id
    Chave Estrangeira: bucket_id → buckets(id)
    Políticas RLS: Qualquer pessoa pode inserir fotos de família; usuários autenticados podem visualizar; apenas devAdmin pode excluir ou atualizar.

Schema: outros

  • estados
    Chave Primária: id
    Políticas RLS: Acesso total para devAdmin; leitura para usuários autenticados.
  • etapas (stages)
    Chave Primária: id
    Políticas RLS: Acesso total para devAdmin e Founder; leitura para Guardião, usuários autenticados e visualizadores.
  • guardiões_tenda (tenda_guardioes)
    Chave Primária: id
    Chaves Estrangeiras: tenda_id → tendas(id), profile_id → users_profiles(id)
    Políticas RLS: Acesso total para devAdmin e Founder.

Papéis de Usuário

  • devAdmin: Administrador técnico, com acesso total a todas as tabelas e configurações. Pode gerenciar usuários, papéis e configurações da plataforma. Pode contornar restrições RLS.
  • Founder: Fundador ou super-administrador, com acesso total semelhante ao devAdmin, mas geralmente reservado para fundadores não técnicos.
  • Guardião: Líder de grupo ou tenda. Tem acesso de leitura à maioria das tabelas e pode gerenciar dados do seu grupo/tenda.
  • Visualizador: Usuário somente leitura. Pode visualizar dados públicos, mas não pode modificar nada.
  • Usuário: Usuário autenticado comum. Pode visualizar dados relevantes e atualizar seu próprio perfil, mas não pode modificar dados de outros usuários.
  • admin: Administrador geral, com permissões intermediárias. Pode gerenciar alguns dados e usuários, dependendo das políticas RLS.
  • service_role: Papéis especiais para operações de serviço, como líderes de equipe.

Próximos Passos: Funções de Serviço e Permissões Potenciais

Para tornar o controle de acesso ainda mais flexível e detalhado, recomenda-se integrar as funções de serviço (como líderes de equipe) às políticas de segurança (RLS). Isso permitirá que líderes e participantes de equipes tenham permissões específicas para gerenciar e visualizar dados relacionados à sua equipe ou área de atuação.

  • Líder de Equipe Consagrados: Gerenciar e visualizar dados da equipe Consagrados; aprovar ou atualizar informações dos membros.
  • Líder de Equipe Tsedaka: Gerenciar e visualizar dados da equipe Tsedaka; aprovar ou atualizar informações dos membros.
  • Padre Paroquial: Visualizar e gerenciar dados da paróquia; supervisionar eventos e membros paroquiais.
  • Fundador: Permissões em toda a plataforma, semelhante ao super-admin; supervisionar todas as equipes e dados.
  • Participante de Equipe Berçário/Creche: Visualizar e participar das atividades da equipe Berçário/Creche; permissões limitadas de edição.
  • Participante de Equipe Ventania: Visualizar e participar das atividades da equipe Ventania; permissões limitadas de edição.
  • Líder de Equipe Berçário/Creche: Gerenciar e visualizar dados da equipe Berçário/Creche; aprovar ou atualizar informações dos membros.
  • Líder de Equipe GAA: Gerenciar e visualizar dados da equipe GAA; aprovar ou atualizar informações dos membros.
  • Guardião: Supervisionar e gerenciar um grupo ou tenda; visualizar e atualizar dados do grupo.
  • Cofundador: Permissões em toda a plataforma, semelhante ao Fundador; supervisionar todas as equipes e dados.
  • Padre: Visualizar e gerenciar dados de paroquianos e eventos, limitado ao escopo da paróquia.
  • Participante de Equipe de Trabalho: Visualizar e participar das atividades da Equipe de Trabalho; permissões limitadas de edição.
  • Treinador de Equipe de Trabalho: Gerenciar e visualizar dados da Equipe de Trabalho; supervisionar treinamentos e desenvolvimento da equipe.
